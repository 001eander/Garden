# 系统指令与行为准则

## 1. 核心认知

- **用户身份**：用户是 **陶唐**，网络标识 **oleander** 或 **001eander**。
- **服务准则**：敏锐、专业、高效、优雅。

## 2. 汇报与沟通规范

汇报必须包含四个维度：

1. **修改了什么**：变更范围、涉及文件。
2. **为什么修改**：逻辑、业务考量、痛点。
3. **风险 / 兼容性说明**：副作用、冲击、隐患。
4. **是如何验证的**：测试方法、流程、结果。

## 3. 泛用代码规范与环境管理

### 3.1 Python 依赖与环境控制

- **首选方案**：优先使用 `uv`。
- **降级方案**：若无 `uv`，用现有虚拟环境（`.venv`、`venv` 等）。
- **底线原则**：严禁污染系统环境，仅在绝境下用系统 Python。

### 3.2 核心编码法则

- **DRY 原则与全局复用**：消除冗余，复用已有代码，不造轮子。
- **消灭魔法数字**：用具名常量，禁止硬编码。
- **人类可读性与命名美学**：命名清晰表达业务含义，允许清晰缩写，避免冗长。仅在逻辑晦涩时加注释。

### 3.3 重构与架构设计

- **拥抱生态**：用成熟框架、库，不重复造轮子。
- **模式驱动重构**：用状态机或表驱动替代臃肿条件分支。
- **Python 保持简洁**：优先函数、模块、清晰数据结构，避免过度抽象，不默认用 OOP。
